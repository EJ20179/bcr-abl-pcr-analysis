## 2. 배경 및 문제 정의

생거 시퀀싱(Sanger Sequencing) 결과물인 ABI 파일을 분석하려면 기존에는 외부 상용 SW(SeqMan, CLC, Benchling 등)나 인터넷 연결이 필요한 웹 서비스를 이용해야 했다.

**핵심 문제점:**

- 보안·규정상 환자 검체 데이터를 외부 서버에 전송 불가
- 인터넷 불안정 환경(내부망 전용 PC)에서 분석 불가
- 팀 내 공통된 레퍼런스 서열 관리 체계 부재
- Forward / Reverse 쌍 분석 결과를 한눈에 비교하기 어려움

---

## 3. 비즈니스 목표

1. **완전 오프라인** 환경에서 ABI 파일 파싱 → 레퍼런스 정렬 → 변이 감지 전 과정 처리
2. 분자유전학팀 워크플로우에 맞게 Word 형식 레퍼런스 문서(기존 자산)를 그대로 활용
3. F/R 쌍 자동 매칭으로 양방향 확인(bilaterally confirmed) 변이만 신뢰 결과로 강조
4. 팀원 누구나 브라우저 하나로 즉시 사용 — 설치·환경 설정 불필요

## 5. 비즈니스 요구사항

### BR-01 오프라인 완전 동작
인터넷 연결 없이 로컬 브라우저만으로 모든 기능이 동작해야 한다.  
외부 CDN, API 호출, 서버 사이드 처리 없이 단일 HTML 파일로 배포 가능해야 한다.

### BR-02 ABI 파일 파싱
Applied Biosystems ABI/ABIF 바이너리 포맷(.ab1)을 브라우저에서 직접 파싱해야 한다.

### BR-03 레퍼런스 서열 다중 입력
- 텍스트 직접 입력
- FASTA 파일
- Word(.docx) 파일 — 팀 내 기존 레퍼런스 문서 자산 재활용

### BR-04 F/R 쌍 자동 매칭
파일명 패턴으로 Forward/Reverse 프라이머 쌍을 자동 인식하고 동일 Exon 단위로 묶어야 한다.

### BR-05 변이 감지 및 검증
SNV·Insertion·Deletion을 감지하고, F와 R 모두에서 같은 변이가 확인된 경우(bilateral confirmed) 별도 강조 표시해야 한다.

### BR-06 레퍼런스 라이브러리
자주 쓰는 레퍼런스 서열을 브라우저 로컬 스토리지에 저장·불러오기·버전 관리할 수 있어야 한다.

### BR-07 결과 내보내기
- 정렬 서열: FASTA 파일 다운로드
- 변이 목록: CSV 파일 다운로드 (HGVS 표기 포함)
- 레퍼런스 라이브러리: JSON 내보내기/가져오기

## 7. 제약 및 전제조건

- 브라우저: Chrome 90+ / Edge 90+ 이상 (Canvas API, DataView, File API 필요)
- 단일 HTML 파일 유지 — 별도 백엔드·DB 서버 불가
- 환자 데이터는 로컬에서만 처리, 외부 전송 절대 불가
- Word 파일 파싱은 ABIF 형식으로 작성된 팀 내 레퍼런스 문서 포맷 기준
